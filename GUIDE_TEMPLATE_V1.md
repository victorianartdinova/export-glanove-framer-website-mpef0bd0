## 0. Принципы шаблона

- **Skim-able first.** Читатель должен схватить суть гайда за 20 секунд по TOC, hero и заголовкам шагов.
- **Один шаг — один экран.** Каждый шаг визуально замкнут, в нём есть «что делать» и «что попробовать сегодня».
- **Большая нумерация шагов.** 01 → 02 → 03 → 04 (опц. 05-06). Цифра как самостоятельный визуальный элемент.
- **Try-this box в каждом шаге.** Гайд без действия = эссе. Каждый шаг заканчивается конкретным «сделай это на этой неделе».
- **Ловушки видны.** В каждом шаге — 2-3 типичные ошибки в trap-card.
- **Цитата между шагами.** Дать паузу, показать живого человека, повысить доверие.
- **CTA в каждой смысловой секции** + один Hard в финале.
- **Анти-AI тон.** Без «эффективных решений», «уникальных подходов», «индивидуально под клиента». Без длинных тире `—` в заголовках.

## 2. TOC (Table of Contents)

### Desktop

- **Sticky column слева**, ширина 220-260px, top-offset под top-nav
- Список всех шагов с номерами: `01 · Название шага`
- Текущий активный шаг — accent-цвет + слева тонкая полоска (3px)
- Scroll-spy: при прокрутке к новому шагу TOC автоматически переключает active
- Hover на пункте — лёгкий fade + cursor pointer
- Клик — плавный scroll (`scroll-behavior: smooth`) с offset под top-nav

### Mobile (< 810px)

- TOC становится **collapsible accordion** прямо под top-nav
- Закрытое состояние — одна строка «Содержание гайда (6)» + chevron
- Раскрытое — вертикальный список шагов
- Тап на шаг → плавный scroll → автоматически закрыть accordion

### Что должно быть в TOC

- Только главные шаги (01-06), без подпунктов
- Без секций Intro / Key Takeaway / CTA / Related — это не контент
- Опционально: внизу TOC «Связанные гайды» одной строкой (опц.)

## 3. Шаги (главный паттерн)

### 3.1 Оптимальное количество

| Тип гайда | Кол-во шагов | Пример |
|-----------|--------------|--------|
| Тактический (как сделать X) | **4** (классика по эталону) | «Как застройщику вести Telegram» |
| Стратегический (что делать в году/сегменте) | **5-6** | «Стратегия маркетинга для девелопера 2026» |
| Обзорный (выбор канала / инструмента) | **4-5** | «Где брокеру искать клиентов 2026» |
| Методологический (Framework) | **3-5** компонентов системы | «Сквозная аналитика для застройщика» |

**Меньше 4 шагов** — это не гайд, это пост. **Больше 6** — гайд распадается на 2.

### 3.2 Анатомия одного шага

```
┌────────────────────────────────────────────────────────────┐
│ ┌────┐                                                     │
│ │ 01 │  Шаг 01 · Соберите ядро аудитории                  │
│ │    │  Цель: понять кого вы зовёте в канал и через что    │
│ └────┘  они туда приходят                                  │
│                                                             │
│  ┌────────────────────────┬───────────────────────────┐    │
│  │ Что делать             │ Главные ловушки           │    │
│  │ ① описание шага 1      │ ⚠ ловушка 1              │    │
│  │ ② описание шага 2      │ ⚠ ловушка 2              │    │
│  │ ③ описание шага 3      │ ⚠ ловушка 3              │    │
│  │ ④ описание шага 4      │                           │    │
│  └────────────────────────┴───────────────────────────┘    │
│                                                             │
│  [Опционально: SVG-схема / скрин / mermaid внутри шага]    │
│                                                             │
│  ┌──── Try-this box ───────────────────────────────────┐   │
│  │ Что попробовать на этой неделе                      │   │
│  │ Конкретное действие в 1-2 предложениях              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Soft CTA: «Так мы это сделали в кейсе [клиент] →»         │
└────────────────────────────────────────────────────────────┘
```

### 3.3 Большая нумерация

- Стиль: Fraunces/serif или Suisse Bold, clamp(120px, 18vw, 280px)
- Цвет: accent под этап (можно одинаковый для всех, можно градиент 01→04)
- Позиция: слева от заголовка шага, baseline-aligned с H2

### 3.4 Двухколоночник «Что делать / Главные ловушки»

**Левая колонка — «Что делать»:**
- Заголовок маленький, заглавными
- Нумерованный список 1-4 с цветными кружками-номерами
- Каждый пункт — 1 фраза действия (не описание)

**Правая колонка — «Главные ловушки»:**
- 2-3 trap-card
- Каждая trap-card: emoji-индикатор (⚠ / 🚫 / 💀) + название ловушки + 1 строка пояснения
- Фон карточки — субтёмный, border-left accent красного/оранжевого

На mobile — стек, сначала «Что делать», потом «Ловушки».

### 3.5 Try-this box (обязательный в каждом шаге)

- Цветная карточка с border-left 4px accent
- Заголовок: «Что попробовать на этой неделе» / «Action item» / «Сделайте сегодня»
- Текст: одно конкретное действие на 1-2 предложения
- Без CTA внутри (чтобы не сливалось с Soft CTA шага)

Примеры action item:
- «Запишите 5 запросов, по которым ваши клиенты ищут квартиру в Google»
- «Откройте Telegram Ads, поставьте бюджет 3 000 ₽/день на тестовую кампанию»
- «Соберите топ-5 конкурентов в одну таблицу: канал, бюджет, формат»

### 3.6 Soft CTA в шаге

После try-this box или внутри шага — 1 строка-ссылка:
- «Так делали в кейсе [клиент]» → case page
- «Подробный разбор техники → [гайд]» → другой guide
- «Подписаться на канал, там разбираем такие штуки в эфире»

## 4. Визуальные блоки

### 4.1 Какие типы блоков допустимы

| Блок | Когда использовать | Где |
|------|--------------------|-----|
| **Callout / Info box** | Важное замечание, не ломающее поток | Внутри шага |
| **Цитата в рамке** | Между шагами, чтобы дать паузу | Между секциями |
| **SVG-схема** | Воронка, путь, древо решений, сравнение | Внутри шага / Intro |
| **Mermaid-диаграмма** | Flowchart, gantt, sequence | Внутри шага |
| **Скрин с разметкой** | Реальный кабинет, CRM, Метрика | Внутри шага |
| **Сравнительная таблица** | Каналы / форматы / бюджеты | Шаг сравнения |
| **График динамики** | Динамика метрики за период | Шаг с цифрами |
| **Инфографика «было/стало»** | Контраст до и после | Intro / Key Takeaway |
| **Чек-лист** | В конце гайда или внутри финального шага | Перед CTA-блоком |

### 4.2 Когда нужны SVG-схемы

**Обязательно SVG / mermaid:**
- В Intro — сравнение «раньше vs теперь» в виде воронок
- В шаге с воронкой / pipeline — flowchart
- В шаге сравнения путей — древо решений
- В шаге планирования (gantt-таймлайн) — горизонтальная диаграмма

**Можно обойтись без схемы:**
- В шаге «что писать» (контентные шаги) — достаточно скринов примеров
- В шаге «как нанимать» — достаточно текста и чек-листа

**Правило:** минимум 1 визуальный элемент на шаг. Без визуала = полотно текста = не Guide.

### 4.3 Callout-боксы

Три типа:
- **Info** (синий border, ℹ) — полезное замечание
- **Warning** (жёлтый/оранжевый, ⚠) — типичная ошибка, риск
- **Success** (зелёный, ✓) — что хорошо работает / best practice

Не более 1-2 callout на шаг. Иначе ломают ритм.

### 4.4 Цитаты между шагами

Между шагами 02-03 или 03-04 — карточка-цитата:
- Большие кавычки сверху
- Текст в Fraunces/serif, увеличенный кегль
- 30-60 слов
- Подпись справа снизу: имя · должность · компания · опционально ссылка на канал
- Фон cream/dark (наследуем от сайта)

Если согласованной цитаты нет — секцию пропускаем. **Не фейк.**

## 5. CTA в Guide

### 5.1 Размещение CTA по секциям

| Секция | Тип CTA | Содержание |
|--------|---------|------------|
| Hero | — (только кнопка «Начать читать» + Soft на канал) | не Hard, ещё не прогрели |
| Intro | — | контекст без действия |
| Шаг 01 | Soft (в Try-this box идёт ссылка на ресурс/кейс) | «Так делали в кейсе X →» |
| Шаг 02 | Soft | «Подробнее в гайде Y →» |
| Шаг 03 | Soft + (опц.) inline-mention продукта | «Под ключ это делает [продукт]» — но без кнопки |
| Шаг 04+ | Может быть mini-Hard | «Заказать аудит → [продукт]» (одной строкой) |
| Цитата | — | не CTA |
| Key Takeaway | Soft | «Подписаться на канал, чтобы не пропустить разбор» |
| Финальный CTA-блок | **Hard, единственный жирный** | «Заказать [продукт] → стартовать» |
| Related cards | Secondary | переход на смежный гайд / кейс |

### 5.2 Финальный CTA-блок (обязательный)

По образцу `KEY TAKEAWAY` из эталона:
- Полноширинная тёмная секция (`bg-ink-900`)
- H2: «Хочу разобрать под мой объект» (или контекстно: «Запустить Telegram-канал под ключ»)
- Подзаголовок 1-2 строки: что внутри услуги
- **Одна** кнопка → product page (Hard)
- Никаких «или подпишись» рядом

### 5.3 Соотношение Soft / Hard

Типичный Guide на 4 шага должен иметь:
- 2-3 Soft CTA в середине
- 1 Hard CTA в финале
- 1 Soft CTA в Hero (подписка на канал)
- 0 CTA в Intro и Цитате

Итого ~5 CTA. Больше — пользователь устанет, страница станет «впаривающей».

## 6. Схемы (когда и какие)

| Тип схемы | Когда использовать | Инструмент |
|-----------|--------------------|------------|
| **Воронка** (показы → лиды → сделки) | Шаг про метрики / Intro «как считать» | Кастомный SVG или mermaid `flowchart LR` |
| **Сравнение двух воронок** «было/стало» | Intro «что изменилось» | Два SVG бок о бок |
| **Путь пользователя** (touchpoints) | Шаг про коммуникации / контент-стратегию | Кастомный SVG-таймлайн |
| **Древо решений** (если X → то Y) | Шаг выбора канала / формата | Mermaid `graph TD` или ручной SVG |
| **Gantt-таймлайн** (что когда запускать) | Шаг 04 «План на N недель» | Mermaid `gantt` |
| **Sequence** (взаимодействия между ролями) | Гайд по процессу команды | Mermaid `sequenceDiagram` |
| **Радар / стрелочная карта** | Framework: компоненты системы | Кастомный SVG |
| **Bar chart / Line chart** | Сравнение бюджетов / метрик / каналов | Chart.js (статичный) или SVG |

**Цветовая палитра схем:** наследуем от сайта (Suisse Intl + ink-900 + cream + accent цвет агентства). Без 3D, без градиентов по диагонали, без drop-shadow.

## 7. Чек-листы (когда добавлять)

### Когда нужен чек-лист

- Гайд тактический и есть набор пунктов для самопроверки (например, «10 вещей в канале застройщика, без которых не работает реклама»)
- Гайд большой и нужно дать дайджест в конце
- Гайд используется как lead-magnet (можно скачать PDF)

### Когда чек-лист НЕ нужен

- Гайд концептуальный / Framework — там лучше итоговая схема
- Гайд короткий (3 шага) — чек-лист дублирует содержание

### Формат

**Inline-чеклист** (в HTML):
```
✓  Прописали ICP (с указанием чека и локации)
✓  Заполнили шапку канала (1 фраза о вас + 1 ссылка)
✓  Подключили автобота для приёма заявок
[...]
```

15-20 пунктов максимум. Иначе становится Spec, а не чек-лист.

**Загружаемый чек-лист** (опционально):
- PDF/Notion-страница
- CTA «Скачать чек-лист» → отдаёт за email (Soft conversion в воронку)
- Подходит для топ-гайдов (приоритет 1-2 по `ARTICLES_MAP_REAL_ESTATE.md`)

### Где в гайде

- Inline — перед финальным CTA-блоком, в финальном шаге или как отдельная секция «Чек-лист»
- Загружаемый — Soft CTA в Hero или в Key Takeaway

## 8. SEO-структура

### Meta-теги

| Тег | Требование | Пример |
|-----|------------|--------|
| `<title>` | 50-60 символов, фокус-кейворд в начале | «Как застройщику вести Telegram в 2026 — гайд ГЛАВНОЕ» |
| `<meta description>` | 150-160 символов, бенефит + цифра + CTA-намёк | «Пошаговый гайд: как застройщику собрать Telegram-канал, заполнить контентом и подключить рекламу. 4 шага и чек-лист на 15 пунктов.» |
| `<meta keywords>` | НЕ нужен (Google игнорирует) | — |
| `<link rel="canonical">` | Обязателен | `https://theglavnoe.com/blog/[slug]/` |

### Структура заголовков

- **Один H1** на странице (в hero, совпадает с `<title>` по смыслу, но может быть длиннее)
- **H2** на каждой смысловой секции: каждый шаг 01-04, Intro, Key Takeaway, CTA, Related
- **H3** внутри шагов, если есть подсекции («Что делать» / «Главные ловушки»)
- **H4** — не злоупотреблять, только если нужно (внутри trap-card нет H4 — title как `<strong>`)

### Open Graph

| Тег | Значение |
|-----|----------|
| `og:title` | Совпадает с `<title>` |
| `og:description` | Совпадает с meta description |
| `og:type` | `article` |
| `og:image` | Отдельный 1200×630, с заголовком гайда крупно. **Не Framer template-изображение.** Лежит в `/blog/[slug]/og.jpg` |
| `og:url` | Полный URL гайда |
| `og:site_name` | «ГЛАВНОЕ» |

### HowTo / FAQ Schema (опционально)

- **HowTo** — если гайд именно пошаговый (4-6 шагов) и каждый шаг — действие. Размечаем шаги.
- **FAQPage** — если в гайде есть отдельная FAQ-секция (3-5 вопросов).

### Внутренние ссылки

В каждом гайде:
- **2-3 ссылки на кейсы** (proof) — обязательно
- **1-2 ссылки на смежные гайды** — обязательно
- **1 ссылка на product page** в финальном CTA — обязательно
- **Опционально:** ссылка на framework-страницу

Анкоры — естественные, не «нажмите сюда».

| Двухколоночник «Что делать / Ловушки» внутри шага | 2 column | стек, сначала «Что делать», потом «Ловушки» |
| SVG-схемы | full width | full width с горизонтальным scroll если содержание не помещается |
| Mermaid-диаграммы | inline | accordion-collapse если длиннее 3 экранов |
| Чек-лист | 2 column (если 15+ пунктов) | 1 column |
| Related cards (3 шт) | grid 1×3 | стек 1×1 |
| CTA-блок финальный | full width centered | full width centered (без изменений) |

Что НЕ меняется:
- Один H1
- Все шаги в порядке (не свернуть в accordion)
- Try-this box в каждом шаге
- Hard CTA в финале

## 10. Author bio + Last updated date

### Author bio — да

В footer гайда (перед related cards):
- Аватар автора или логотип агентства (если общий автор)
- Имя + роль (например, «Виктория, основатель ГЛАВНОЕ»)
- 1-2 строки био
- Ссылки: TG-канал, контакт

Если у гайда несколько авторов — карточка агентства, не персональная.

### Last updated date — да, обязательно

- Размещение: в hero под подзаголовком (тонкий текст «Обновлено: 22 мая 2026»)
- Также в Schema.org Article (`dateModified`)
- При обновлении гайда дату менять — это сигнал Google и доверие читателю

### Reading time — опционально

Рядом с last updated:
- «Чтение: 12 минут» — для гайдов > 6 шагов
- Считается просто: ~200 слов в минуту
- Размещение: hero, тонким серым
