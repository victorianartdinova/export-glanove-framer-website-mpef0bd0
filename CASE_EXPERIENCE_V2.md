# CASE_EXPERIENCE_V2 — кейсы как обучающий опыт

Документ про **ощущение чтения**, не про код. Цель — лучший опыт изучения кейса в нише недвижимости в Рунете.

Опирается на `ai-startup-guide.html` (`/root/ops/products/ai-startup-guide.html`) — берём только **опыт потребления**, не структуру блоков.

---

## 1. Сдвиг парадигмы

| Сейчас (что есть) | Куда идём (V2) |
|---|---|
| Кейс = тёмный лендинг | Кейс = обучающий материал |
| Цель: показать «вот наш результат» | Цель: научить так же. Покупка — побочный эффект пользы |
| Читать 10 мин = тяжело, тревожно | Читать 10-15 мин = приятно, как Notion-страница / журнал |
| Главное достижение = красивая цифра | Главное достижение = инсайт, который человек заберёт |
| Картинки = декор | Картинки = объяснение того что только что прочитано |
| Анимация = oh shiny | Анимация = шаг воронки видно прямо в движении |
| После прочтения: «красиво» | После прочтения: «понял как работает, могу повторить» |

**Принцип:** кейс должен давать пользу даже тому, кто никогда не купит. Это и есть лучшая лидогенерация.

## 2. Цветовая система — paper-style

### Фон
Основной: **#FBF7F1** — тёплый кремовый, как страница хорошей книги или премиум-журнал.
Альтернатива: **#F6F3EE** (чуть холоднее) для wow-блоков.

### Текст
- Заголовки: **#1A1814** (почти чёрный, но тёплый)
- Основной текст: **#2A2723**
- Метатекст / подписи: **#6B6660**

### Акценты (используются только для смысла, не для красоты)
| Цвет | Hex | Когда |
|---|---|---|
| Зелёный мятный | #5CBA8C / gradient #ECFDF5→#D1FAE5 | Положительный инсайт / рост / победа |
| Оранжевый кораллов | #F0805F / gradient #FEF3F0→#FDE2DA | Сложность / ловушка / разочарование |
| Фиолетовый лавандовый | #A087D3 / gradient #F6F3FC→#ECE6F8 | Инсайт / переосмысление / стратегия |
| Жёлтый медовый | #F5C24A | Подсветка ключевой цифры (one-liner) |

### Тёмные блоки — только точечно
Чёрный (#1A1814) фон допустим **только** для wow-блока «итоговая цифра» / завершающего CTA. Не для тела кейса.

### Принцип «paper»
- Тонкая хайрлайн-линия #E5E0D8 между секциями (как линейка в блокноте)
- Шрифт основного текста: Suisse Intl Regular 17-18px, line-height 1.7 (для длинного чтения)
- Заголовки: Suisse Intl SemiBold, не bold-черные
- Ширина текстового блока: max 720px центрированно

## 3. Опыт первой минуты на странице

Что человек видит когда открывает кейс. Хронология первых 60 секунд.

### Секунды 0-3
- Тёплый светлый фон **сразу** (никакого тёмного flash)
- Breadcrumb сверху мелким: `ГЛАВНОЕ → Кейсы → [имя кейса]`
- Большой заголовок-вопрос: не «Как мы получили 6 сделок», а **«Можно ли через Telegram Ads закрыть 6 сделок на 90 млн ₽ за 4 месяца?»**
- Ниже — одна строка субтитра с контекстом: «Премиум-агентство СПб. Бюджет 1 млн ₽. 4 месяца работы.»

**Чувство:** «Ага, это история, а не презентация. Можно почитать.»

### Секунды 3-8
- Появляется **карточка читателя**: «⏱ 12 мин чтения · 📊 4 этапа · 🎯 для девелоперов и брокеров премиум-недвижимости»
- Прогресс-бар сверху (sticky), пока в начале — пустой
- Кнопка «Слушать аудио» если доступно (не сейчас, на будущее)

**Чувство:** «Понятно сколько займёт. Я знаю что покупаю с точки зрения времени.»

### Секунды 8-15
- Под hero — **сводная карточка** одной строкой: 4 ключевые цифры в строку, без пафоса.
- Каждая цифра — это **обещание**: «Ниже я расскажу как это получилось».

**Чувство:** «Ок, цифры есть. Если я готов разобраться — поехали.»

### Секунды 15-30
- Sticky TOC слева (desktop) или collapsed accordion (mobile)
- Первый параграф — это **вопрос-зацепка**: «С чего мы вообще начали? Когда [клиент] пришёл, у него был [проблема]. Это типичная ситуация для…»
- Не «мы провели аудит» — а живой разговор от первого лица

**Чувство:** «Человек со мной разговаривает, не корпоративное письмо.»

### Секунды 30-60
- Появляется **визуальная сценка** — первая мини-иллюстрация (см. раздел 7).
- Сценка не декоративная. Она показывает «вот была ситуация».
- Под сценкой — «Что мы поняли в первую неделю» (3 буллета как открытие, не как процесс)

**Чувство:** «Я уже что-то узнал. Это уже было полезно.»

## 4. Прогресс прохождения

Длинный кейс = инвестиция времени читателя. Он должен чувствовать что движется.

### Sticky progress bar сверху
- Тонкая линия 2px, цвет — мятный gradient (растёт слева направо).
- Заполняется по мере scroll.
- На mobile прячется при scroll вниз (чтобы не мешать), появляется при scroll вверх.

### Чекпоинты в TOC
- Каждый пройденный раздел в TOC становится **зачёркнутым** или с галочкой 🟢.
- Текущий — выделен фоном.
- Будущий — приглушён.

**Цель:** дать чувство «я уже на середине, ещё чуть-чуть».

### Микро-метки внутри текста
- В начале каждого этапа: «**Этап 02 из 04** · Придумали формат» (большая нумерация слева, как в ai-startup).
- В конце этапа: «**До следующего этапа осталось 3 минуты чтения**».

### Финал — «Ты прошёл весь кейс»
- В самом низу — анимированная отметка о прохождении (например, галочка с pulseSoft анимацией).
- Подпись: «Спасибо что дочитал. Если что-то осталось непонятным — напиши, разберём.»
- CTA не «покупай», а «получить бесплатный аудит / задать вопрос».

## 5. Анимации со смыслом (не для красоты)

### Принцип
Анимация **объясняет** что только что прочитано. Если её убрать — смысл не меняется, но понимание медленнее.

### Конкретные анимации для 3 кейсов

#### SPB 15 млн (Лидген Телеграм)
1. **Воронка лида** — после раздела о Telegram Ads. SVG-схема:
   `Telegram Ads баннер → клик → подписка на канал → бот WhatsApp/Telegram → созвон брокера → сделка`
   Анимация: маленький кружок-«лид» (синий) последовательно движется по этим узлам с задержкой 600мс между ними. Останавливается в каждом узле, узел подсвечивается мятным.
   **Что объясняет:** что лид не просто «появился», а прошёл 5 шагов системы.

2. **График бюджета по месяцам** — после раздела о масштабировании. Столбцы вырастают слева направо с задержкой 200мс (CSS keyframe `growBar`). Числа считаются от 0 до финального (`popDot`).
   **Что объясняет:** что бюджет рос постепенно, не сразу 1 млн.

3. **«До vs После»** — в финале. Слева: одинокий брокер с телефоном (статичная иллюстрация в #6B6660), стрелка `→` появляется (`drawPath`), справа: тот же брокер с поток заявок из Telegram (мятный gradient).
   **Что объясняет:** трансформацию работы клиента.

#### AAG ПромоСтраницы
1. **Анатомия статьи на ПромоСтраницах** — после раздела о формате «Диалоги с эстетом». Прокручиваемый mockup статьи (вертикальный telephone-style frame). Внутри — текст «Каким должен быть идеальный дом в центре Петербурга», постепенно scrolls сверху вниз (CSS animation 8s linear). Сбоку — «глаз читателя» который следит за прокруткой.
   **Что объясняет:** что статья читается долго (3 мин), и за это время идёт прогрев.

2. **Воронка ПромоСтраниц** — отдельная SVG-схема.
   `Промостатья → дочитывание (счётчик растёт от 0 до 1115) → переход на сайт (счётчик до 602) → дозабор через Метрику/ретаргетинг → лид`
   **Что объясняет:** что не все читатели становятся лидами, и это нормально.

3. **Сравнение CPM** — 3 столбца: «Контекст 2500₽ / Баннер TG 1231₽ / ПромоСтраницы 310₽». Высота столбцов рисуется снизу вверх (`growBar`). Последний столбец помечен мятным с подписью «в 8 раз дешевле».
   **Что объясняет:** математику окупаемости ПромоСтраниц.

#### Regardis (премиум TG Ads)
1. **Креативная воронка Telegram Ads** — последовательность из 12 баннеров мелких карточек. Появляются по очереди (`floatY`). На последней — пометка «оставили 5 из 12 после теста».
   **Что объясняет:** что креативы тестируются, а не «нагенерили и запустили».

2. **Карта премиум-сегмента** — карта Москвы (упрощённая SVG), на ней появляются точки-зоны (Хамовники, Патриаршие, Якиманка...) с числами «50 заявок / 9684₽ CPL / 30 млн ₽ ср.чек». Анимация: точки появляются `popDot`.
   **Что объясняет:** что премиум-аудитория — географически концентрирована и её можно таргетировать.

3. **Снижение CPL по неделям** — линейный график за апрель→май (CPL: 11k → 9.6k → 6.8k). Линия рисуется слева направо (`drawPath`). В точке «-30%» появляется callout-bubble с пояснением.
   **Что объясняет:** что оптимизация — это процесс, а не однократное действие.

### Технические принципы анимаций
- **`prefers-reduced-motion: reduce`** — статичные fallback.
- Длительность одной анимации: 600-1500мс, не дольше.
- Триггер: `IntersectionObserver` (запускается когда блок попадает в viewport).
- Никаких автоплеев на каждом scroll — анимация играет один раз, потом статика.
- Кадр финального состояния всегда читабелен без анимации (на случай если анимация не сработала).

## 6. Персонажи — визуальные помощники

### Зачем
Текст про «лид» абстрактен. Лицо брокера/девелопера/покупателя — конкретно. Между блоками появляется персонаж и говорит фразу — это удерживает внимание.

### Кастинг персонажей

| Персонаж | Образ | Когда появляется |
|---|---|---|
| **Виктория** | реальный портрет (фото с сайта в circle-frame), tone «опытный гид» | На переходах между этапами: «Дальше расскажу что мы делали…» |
| **Брокер** | стилизованная иллюстрация (line-art, тёплые цвета) — мужчина 30+ в рубашке, без галстука | На воронке: «Я обрабатываю заявку за 2 часа» |
| **Девелопер-клиент** | line-art иллюстрация — деловой портрет, без конкретных черт | На контексте: «Когда я пришёл к ним, у меня было…» |
| **Покупатель премиум** | line-art — женщина / мужчина с телефоном, спокойная поза | На воронке: «Я увидел баннер в Telegram и…» |
| **Объект (ЖК)** | стилизованная карточка-illustration с фасадом дома, окнами, цифрой этажности | На контексте: «Объект клиента — AAG/CORE-XP/Regardis…» |

### Принципы изображения
- НЕ мультяшный стиль, НЕ stock-фото. Что-то между Stripe-иллюстрациями и Notion-doodles.
- Line-art с заливкой пастельным акцентом (мятный/коралловый/лавандовый).
- Размер 80-120px (не больше иконки).
- Появляются **между разделами** или на полях, не в центре чтения.
- НЕ говорят слишком много. Максимум одно предложение в speech-bubble.

### Где появляются
- Hero: круглый портрет Виктории справа от заголовка с подписью «Виктория Нартдинова · founder ГЛАВНОЕ. Этот кейс — мой».
- Переходы между этапами: маленький Виктория-аватар + фраза-связка («Дальше пошли в Telegram Ads. Это самое интересное.»).
- На воронке: иконки персонажей в каждом узле воронки (брокер, покупатель, объект).
- В разделе «Главные ошибки на этапе» (trap-card): иллюстрация «грабли» или «турникет» — мягкая, не пугающая.
- В финале: Виктория с фразой «Если у тебя похожая задача — давай разберём».

## 7. Wow-моменты — что после каждого этапа

После каждого этапа (Изучили / Придумали / Запустили / Получили) пользователь должен получить **что-то осязаемое**, помимо текста.

### Этап 01 «Изучили» → wow-инсайт
- В конце раздела — выделенный callout-блок (paper-card, фоновый мятный gradient).
- Заголовок: «Что мы поняли в первую неделю».
- 3 буллета — это **открытия**, не «процесс»:
  - ✅ Премиум-аудитория ловится через 5 конкретных каналов в Telegram, а не через таргет «премиум»
  - ✅ CRM клиента не успевает обрабатывать → нужен бот
  - ✅ Брокеры не готовы к 50 лидов/мес → нужно обучить
- Подпись: «Это уже стоит того, что ты прочитал».

### Этап 02 «Придумали» → визуальная схема
- Большая SVG-схема концепции (не словами).
- Например для SPB: схема «фильтр премиум-лидов» — на входе 1000 лидов, фильтры по чеку / готовности / срочности, на выходе 50 квал.
- Анимация: маленькие кружки-лиды падают сверху, фильтруются, оставшиеся вылетают вправо.
- Подпись: «Это и есть концепция. Можешь её скопировать.»

### Этап 03 «Запустили» → live-метрики
- Карточка с **анимированными счётчиками**:
  - «260 обращений за 4 месяца» (счётчик от 0 до 260, 2 секунды)
  - «5113 ₽ средняя стоимость обращения» (счётчик)
  - «19 973 ₽ квал. клиент» (счётчик)
- Под каждой цифрой — мелкая подпись «vs рынок 30 000₽» (для сравнения).
- Эффект: ощущение «всё это реально, не выдуманный кейс».

### Этап 04 «Получили» → «До vs После» сценка
- Слева: иллюстрация клиента до — пустой Telegram-канал, печальный брокер с одним сообщением.
- Стрелка `→` с подписью «4 месяца работы».
- Справа: тот же брокер с потоком сообщений, заполненный канал, цифра «6 сделок × 15 млн».
- Сценка статичная, но появляется по scroll (с fade-in).

### Финал → «Что забрать с собой»
- Чёрный callout-блок (единственный тёмный блок в кейсе) — с **5 takeaways**.
- Это не «купи у нас», это «забери эти 5 правил себе»:
  1. Не запускай Telegram Ads без бота для обработки
  2. CPL квал. лида — не CPL подписки
  3. Тестируй 4 оффера параллельно
  4. Меняй креативы раз в неделю
  5. Бюджет растёт от первых результатов, не наоборот
- Под этим — мягкий CTA: «Если хочешь применить это к своему продукту — напиши».

## 8. Маленькие визуальные сценки между разделами

### Что это
Между этапами кейса вставляется **микро-сценка** (1 секция, 200-300px высоты, со своим фоном-карточкой). Она не несёт критической информации — она даёт **отдых глазам и обозначает переход**.

### Примеры сценок

**Между Этап 01 и 02:**
- Сценка «Идея загорается»: лампочка-illustration с мягким glow-эффектом (CSS `pulseSoft`).
- Текст под лампочкой: «И тогда мы поняли…» (cliffhanger).
- Фон карточки: фиолетовый gradient (#F6F3FC→#ECE6F8).

**Между Этап 02 и 03:**
- Сценка «Запуск»: маленькая ракета (line-art) с траекторией (SVG path с `drawPath`).
- Текст: «Поехали.»
- Фон: мятный gradient (#ECFDF5→#D1FAE5).

**Между Этап 03 и 04:**
- Сценка «Подсчёт результатов»: калькулятор или абак.
- Текст: «Через 4 месяца у нас на руках было…»
- Фон: жёлтый медовый.

### Принципы
- **Не больше 200-300px высоты** (легко проскроллить).
- **Без CTA** (это просто пауза).
- **Без обязательного контента** (если читатель прокрутил мимо — он ничего не потерял).
- Должны быть **разные** по типу (лампочка / ракета / калькулятор / зеркало / сертификат / часы). Не использовать одну и ту же иллюстрацию.

## 9. Что взять из AI Startup и что НЕ брать

### Берём ✅
1. **Paper-фон #FBF7F1** + тёмный текст #1A1814 — это основа.
2. **Анимации `drawPath` / `popDot` / `pulseSoft` / `floatY`** — для смысловых иллюстраций.
3. **Trap-card** (ловушки) — превращаем в «Главные ошибки на этом этапе» — самая ценная инфа для читателя.
4. **TLDR в начале** — карточка «12 мин чтения · 4 этапа · что узнаешь».
5. **Большая нумерация шагов** — слева крупным шрифтом «01 / 02 / 03».
6. **Двухколоночная вёрстка для блока «что делали + почему сработало»**.
7. **Sticky TOC** слева на desktop.
8. **Прогресс-бар сверху**.
9. **Финальная плашка «Что забрать с собой»** — список takeaways.

### НЕ берём ❌
1. **Зелёная палитра как доминанта** — это AI-стартапы. Для недвижки доминанта — тёплая нейтральная (paper), а зелёный/коралловый/лавандовый/жёлтый — только как акценты по смыслу.
2. **Терминология AI Startup** (CAC / LTV / MVP / pivot) — заменяется на язык недвижки (CPL квал. / средний чек / сделка / закрытие).
3. **Длина AI Startup (45 мин чтения)** — для кейсов 10-15 мин макс. Премиум-девелопер не будет читать 45 минут.
4. **Скриншоты Cursor / GitHub UI** — заменяются на скриншоты Метрики / Яндекс ПромоСтраниц / Telegram-канала / Amo CRM.
5. **«Длинный путь vs короткий путь»** — у нас 4 этапа последовательны, без выбора пути.

## 11. Чек-лист «Когда кейс готов»

Каждый кейс должен пройти:

### Опыт чтения
- [ ] Фон тёплый светлый (#FBF7F1), тёмные блоки только в финальном takeaway
- [ ] Текст читается длинно без напряжения (line-height 1.7, max-width 720px)
- [ ] Sticky прогресс-бар сверху работает
- [ ] TOC слева на desktop / accordion на mobile, с чекпоинтами

### Содержание
- [ ] В hero — вопрос-зацепка, не «вот наш кейс»
- [ ] Сводная карточка-обещание под hero (4 цифры одной строкой)
- [ ] 4 этапа с большой нумерацией
- [ ] В каждом этапе: что делали + что не сработало + что поняли
- [ ] Раздел «Главные ошибки» (trap-card стиль) минимум в одном из этапов
- [ ] Минимум 1 анимированная схема на кейс (воронка / график / процесс)
- [ ] Минимум 3 wow-блока (по одному на 2-3 этапа)
- [ ] Между этапами — сценки-переходы (минимум 2)

### Персонажи
- [ ] Виктория появляется в hero и в финале (как минимум)
- [ ] Минимум 1 не-Виктория персонаж (брокер / покупатель / клиент)
- [ ] Персонажи — не stock-фото

### Финал
- [ ] Раздел «Что забрать с собой» с 5 takeaways
- [ ] Soft CTA («задать вопрос» / «получить аудит»), не «купи»
- [ ] Анимированная «галочка прохождения»

### Технически
- [ ] `prefers-reduced-motion` fallback для всех анимаций
- [ ] SVG-схемы вектор, не PNG
- [ ] Mobile-структура: TOC accordion, сценки уменьшены, анимации сокращены
- [ ] Время загрузки < 2 сек cold
- [ ] Schema.org Article + BreadcrumbList

### Эмоциональный тест
- [ ] Прочитал бы я это до конца, если бы не работал над этим?
- [ ] Получил бы я пользу, если бы никогда не купил?
- [ ] Захотелось бы поделиться с коллегой?

## 13. Что НЕ делать в V2

- Не добавлять «дисклеймер» / «политика конфиденциальности» в тело кейса (это для футера)
- Не пихать NDA-плашки (это не доверие, это барьер)
- Не использовать stock-фото из shutterstock-стиля
- Не делать инфографику в стиле «корпоративный PowerPoint» (3D-стрелки, gradient backgrounds типа neon)
- Не использовать слова «синергия», «диджитал», «омниканальность» в первом предложении
- Не делать длиннее 15 минут чтения

## 14. Следующий шаг

После одобрения этого документа:
1. Сделать **дизайн-токены** (CSS-переменные для палитры paper-style)
2. Подготовить **2-3 SVG-сценки** (лампочка, ракета, калькулятор) — переиспользуемые
3. Подготовить **3-4 анимированных мини-компонента** (воронка, счётчик, growBar) — переиспользуемые
4. Подготовить **набор персонажей** (5 line-art иллюстраций) — переиспользуемые
5. Переделать SPB как образец V2 (он эталон, V1 → V2)
6. Прогнать AAG и Regardis по тому же образцу

Без переиспользуемых компонентов каждый кейс будет занимать дни. С компонентами — каждый кейс становится «контентный + лёгкая верстка» (2-3 часа).
