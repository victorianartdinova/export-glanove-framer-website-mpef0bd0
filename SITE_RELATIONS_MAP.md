# SITE_RELATIONS_MAP

Концептуальная карта связей контента сайта `theglavnoe.com`. **НЕ реальные ссылки** — описание того, что с чем должно быть связано после интеграции во Framer.

Опорные документы:
- `CONTENT_ARCHITECTURE_V1.md` — типы страниц и принципы связей (раздел 5)
- `CASE_TEMPLATE_V1.md` — структура кейса, привязка CTA к продуктам
- `ARTICLES_MAP_REAL_ESTATE.md` — карта статей и их продукты
- `/root/ops/CASE_SOURCES_OF_TRUTH.md` — источники цифр для кейсов

Обозначения:
- **Primary CTA** — главное действие (Hard, ведёт к заявке)
- **Secondary CTA** — альтернативный путь (на другой кейс / гайд)
- **Soft CTA** — подписка на канал / переход в блог
- → @ksandrbloger — Telegram CEO (Hard-конверсия)

## ЧАСТЬ IV. ПРАВИЛА ЦЕЛОСТНОСТИ

При публикации/обновлении любой страницы:

1. **Product без кейса** = тревога. Минимум 1 кейс должен быть привязан. Если нет — placeholder «Готовим кейс к публикации» + ссылка на статью-методику.
2. **Case без основного продукта** = брак. Случай не выпускаем без `primary_product`.
3. **Guide без 2-3 кейсов в ссылках** = эссе, не Guide. Возвращаем на доработку.
4. **Case без СМИ-источника** = только внутренние цифры из CRM/рекламного кабинета. Без внешнего пруфа — это нормально, но в чек-листе CASE_TEMPLATE отмечаем.
5. **Цикл «продукт → кейс → продукт»** должен замыкаться. Кликнул на кейс с product page, увидел доказательство, в финале кейса вернулся на тот же или похожий продукт.
