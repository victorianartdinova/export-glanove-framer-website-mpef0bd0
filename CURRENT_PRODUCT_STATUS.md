## Что уже проверено

### Гипотезы и их результат

1. **«В HTML отсутствует разметка партнёров и pre-footer»** — отвергнуто. Строки `Совсем скоро / Карта сайта / hello@glavnoe` есть во всех 5 HTML; партнёры приходят через JS.
2. **«4 страницы не дорендериваются из-за hydration mismatch»** — подтверждено для исходного состояния (slug=`product-statii` в pathVariables давал h=900px и React error #405). Лечится подменой routeId на static-маршрут.
3. **«collectionItemId одинаковый → CMS не различает страницы»** — подтверждено. У всех 5 HTML был `ooRWNc0pX` и `routeId: m1M5Lok7C` (CMS-route Avito). Удаление CMS-полей + смена routeId решает рендер, но даёт другую страницу (v2-chunk).
4. **«Подмена slug в pathVariables на свой»** — НЕ работает, даёт 404 Error.
5. **«Создать per-product копии chunk A (qE2...CquAwsV0.mjs) с подменёнными текстами + переписать ссылку в HTML»** — НЕ работает: `script_main.B9JEDTsv.mjs` хардкодит путь к chunk A через свою router-таблицу, ссылка в HTML им игнорируется.
6. **«Подмена routeId на static-маршрут v2-чанка»** — РАБОТАЕТ для рендеринга, но v2-чанки содержат СОБСТВЕННУЮ структуру страницы (другие тарифы, белый процесс, 4-ячеечный результат), не клон Avito.
7. **«Запустить 4 параллельных general-purpose агента на разбор HTML и подмену»** — НЕ работает: агенты зависают в анализе скриншотов и не доходят до Edit (440+ сек, 44 tool uses, файл не изменён).

### Подходы, которые НЕ НУЖНО повторять

- Создавать суффиксированные копии `qE2...{slug}.mjs` / `V6MP5...{slug}.mjs` и подменять ссылки в HTML.
- Менять `&quot;BDn1NDrfn&quot;:&quot;авито-адс&quot;` на свой slug в pathVariables.
- Удалять `data-framer-hydrate-v2` атрибут на 4 страницах целиком — даёт SSR, но без партнёров.
- Запускать general-purpose агентов с задачей «разобрать HTML и подменить».
