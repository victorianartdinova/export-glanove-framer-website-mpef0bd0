# VISUAL_STYLE_REFERENCE — стилевой ориентир для кейсов

Дополнение к `CASE_EXPERIENCE_V2.md`. Конкретный визуальный язык, который ловим в кейсах недвижимости.

Референс: Glow Design Agency × Product Fruits ([Behance](https://www.behance.net/gallery/245072841/AI-Product-Adoption-Platform-Product-Fruits) · [productfruits.com](https://productfruits.com))

---

## 1. Главное настроение

**Friendly premium**. Между Notion-страницей и анимационным онбордингом Stripe/Linear. Серьёзно, но не корпоративно. Доступно, но не игрушечно.

Не Berghain. Не Apple-keynote. Не нейтрально-корпоративный SaaS.

Это про **«меня учат, и это приятно»**.

---

## 2. Палитра

### База
- Фон: **#FFFCF7** (молочный, чуть теплее чисто-белого, paper-feel без зернистости)
- Текст основной: **#1A1814**
- Текст вторичный: **#5C5750**
- Линии-разделители: **#EFEAE0**

### Акценты по смыслу (используем РЕДКО, не везде)
- 🟠 Главный акцент: **#FF6B35** (заимствуем у Product Fruits) — для CTA, для самой главной цифры, для подсветки в финале
- 🟢 Зелёный мятный: **#5CBA8C** + gradient `#ECFDF5 → #D1FAE5` — для wow-блоков «что получилось» / положительные инсайты
- 🟣 Лавандовый: **#A087D3** + gradient `#F6F3FC → #ECE6F8` — для «что мы поняли» / инсайт-блоков
- 🟡 Медовый: **#F5C24A** — подсветка single-line key facts
- 🔴 Коралловый: **#F0805F** + gradient `#FEF3F0 → #FDE2DA` — для «ловушек» (trap-card)

### Принцип
**Один кейс — один доминантный акцент** (например, SPB = оранжевый Product Fruits для Telegram-связки, AAG = лавандовый для «думающего» формата, Regardis = зелёный для премиум-роста). Остальные акценты по смыслу, не для разнообразия.

## 3. Типографика

### Шрифт
**Suisse Intl** уже стоит на сайте — используем его (Regular для тела, SemiBold для заголовков). Альтернатива fallback: Manrope / DM Sans / system sans-serif.

### Иерархия
| Роль | Размер desktop | Размер mobile | Вес | Цвет |
|---|---|---|---|---|
| H1 (hero) | 48-56px | 32-38px | SemiBold | #1A1814 |
| H2 (этап) | 32-40px | 24-28px | SemiBold | #1A1814 |
| H3 (подраздел) | 22-26px | 18-20px | SemiBold | #1A1814 |
| Body | 17-18px | 16-17px | Regular | #2A2723 |
| Meta / caption | 13-14px | 12-13px | Regular | #5C5750 |
| Large number (метрика) | 64-88px | 40-56px | SemiBold | акцентный цвет |

### Принцип
- `line-height: 1.65` для тела (длинное чтение)
- `letter-spacing: -0.01em` для заголовков (плотнее)
- Жирность для эмфазы, не курсив
- Никаких ВСЁ КАПСОМ кроме редких лейблов «01 ЭТАП»

## 4. Иллюстрации — стиль

### Принципы
- **Flat 2D + occasional 3D-element-accent** (как у Product Fruits — плоские иконки + один 3D-фрукт-маскот)
- **Line-art фон + flat-fill акцент** (контур серый, заливка одним из акцентных цветов)
- **Скруглённые формы** — никаких острых углов
- **Тёплые тона** — даже синий должен быть тёплый (если используется)
- **Без stock-фото**, без shutterstock-вайба

### Что рисуем
- 🏢 **Объект недвижимости** — упрощённый фасад дома (3-4 окна, дверь, без деталей). Один на каждый кейс.
- 👤 **Брокер** — line-art figure с телефоном
- 👤 **Покупатель** — line-art figure с задумчивым видом
- 👤 **Девелопер-клиент** — line-art figure в деловом контексте
- 📱 **Телефон с приложением** — упрощённый mockup, видно Telegram/ПромоСтраницы/CRM
- 📊 **Графики** — простые столбики/линии, BIG NUMBERS, никаких 3D bar charts

### Маскот для всего сайта
Один общий «помощник» — **круглая иконка-домик** (как Product Fruits — фрукт). Появляется:
- В TOC слева как помощник навигации
- На переходах между этапами с короткой репликой
- В финале с прощальной фразой

Образ: уютный домик с окошком, тёплой подсветкой внутри, лёгкая улыбка из окон.

## 5. Карточки

### Floating card style
- Border-radius: **24px** (везде одинаковый — это формирует ритм)
- Background: **#FFFFFF** (на молочном фоне страницы карточка яркая)
- Border: **1px solid #EFEAE0** (тонкая линия)
- Shadow: **0 4px 24px rgba(26, 24, 20, 0.04)** — мягкая, едва заметная
- Padding: **32-40px** desktop, **24px** mobile

### Wow-карточки
- Border-radius: **32px** (чуть крупнее обычных)
- Background: **gradient** одного из акцентных пастельных
- Border: нет
- Shadow: чуть глубже `0 8px 32px rgba(26, 24, 20, 0.06)`
- Внутри — иллюстрация + текст-инсайт

### Карточки метрик (4 в ряд)
- Border-radius: **20px**
- Background: **#FBF7F1** (на тон темнее фона страницы — выделяются)
- Top-border 3px одного из акцентных цветов (как stripe)
- Большая цифра 56-64px + подпись 14px

## 6. Анимации — Jitter-style motion

### Принципы Jitter (https://jitter.video)
- Каждый элемент анимирован **на появление**, не постоянно
- Easing — `cubic-bezier(0.22, 1, 0.36, 1)` (out-expo)
- Длительность 600-1000мс
- Sequential delays между элементами в группе (50-150мс)
- **Триггер: IntersectionObserver** (анимация при попадании в viewport)
- **Reduced motion** — статичный fallback

### Конкретные типы анимаций
| Где | Анимация |
|---|---|
| Hero appear | Заголовок: fade + slide up 16px. Subtitle: + 100мс delay. Cards под hero: + 200мс stagger |
| Floating card on scroll | Поднимается на 12px вверх + fade in |
| Метрика-счётчик | Цифра считается от 0 до финальной за 1.2 сек |
| SVG-воронка | Линии рисуются `drawPath`, узлы появляются `popDot` с delay 200мс между ними |
| Маскот-домик при наведении | Лёгкий wiggle (rotation -3° → 3° → 0°, 400мс) |
| Прогресс-бар | Заполняется при scroll (sync с position) |
| Кнопка CTA hover | Background slide эффект (gradient проходит слева направо) |
| Картинки этапа | Fade + scale 0.95 → 1.0, 800мс |

### Что НЕ делаем
- Parallax (на длинных страницах укачивает)
- Авто-карусели (отвлекают от чтения)
- Slide-in каждой строки текста (раздражает на длинных кейсах)
- Bouncing buttons (детский сад)

## 7. Layout

### Container
- max-width: **1080px** (десктоп) — это шире чем 720px текстового блока, чтобы было место для floating-cards справа от текста
- Текстовый блок: **max-width 720px** центрированно
- На больших экранах (≥1440px) — текст 720px по центру, справа/слева могут «всплывать» sticky-элементы (TOC слева, маскот-комментарии справа)

### Spacing
- Между секциями: **80-120px** desktop, **48-64px** mobile
- Между параграфами: **24px** (просторно)
- Между карточками в группе: **16-24px**
- Padding inside cards: **32px** desktop, **24px** mobile

### Sticky-элементы
- TOC слева на ≥1024px (sticky top:88px), на mobile — accordion в самом верху
- Прогресс-бар сверху (sticky top:0, 2px высоты)
- На mobile при scroll вниз: бургер исчезает, прогресс-бар остаётся

## 8. Микро-элементы

### Callout боксы
- Background: pastel gradient (один из 4 акцентных)
- Border-radius: 24px
- Padding: 24-32px
- Иконка слева (emoji допустимо — 🟢 / 💡 / ⚠️ / 📊)
- Заголовок SemiBold + 2-3 буллета

### Цитаты (без цитат клиента — для журналистских вставок «как в СМИ»)
- Border-left: 4px solid акцентного цвета
- Padding-left: 24px
- Italic
- Цитата + источник (сноска)

### Чек-листы
- Каждый пункт: круг 20px с галочкой ✓ внутри (на акцентном фоне), затем текст
- Pass-state (выполнено) — заливка цветом
- Pending — белый фон + контур

### Графики
- **Bar chart**: вертикальные столбики, rounded top (radius 8px), один акцентный + остальные серые
- **Line chart**: один line stroke 3px, точки 8px circle, акцентный цвет
- **Funnel chart**: trapezoids со ступенчатой шириной, каждый этап одним цветом из gradient (тёмнее наверху → светлее внизу)

## 9. Mobile-структура

- TOC: accordion в верху, по клику разворачивается
- Метрики 4-в-ряд → 2×2
- Floating cards уменьшают padding 24px, radius 20px
- Анимации сокращаются: только entrance, без hover (на touch нет hover)
- Маскот-домик размером 40px (вместо 64-80 desktop)
- SVG-схемы (воронка) — переориентируются: вместо horizontal flow → vertical column
- Большие числа уменьшаются: 88px → 56px

## 12. Что взять для реализации

### Готовые ресурсы
- **Шрифт** — Suisse Intl уже стоит ✓
- **Иконки** — Lucide / Phosphor / Tabler (open-source SVG)
- **Иллюстрации** — заказать у иллюстратора (line-art + flat-fill), либо использовать [unDraw](https://undraw.co) с кастомизацией под нашу палитру
- **Маскот-домик** — заказать отдельно (~6-10 поз: статичная / приветствие / wiggle / удивлён / показывает на схему / читает)
- **Анимации** — JS-библиотека Framer Motion (если внутри Framer) или GSAP / Motion One для статичных страниц

### Что разработать
- 1 главный CSS-файл с переменными `--bg / --text / --accent / --radius / --shadow`
- Набор переиспользуемых компонентов: floating-card, metric-card, callout, funnel, chart, mascot-comment
- 4-5 базовых SVG-illustrations + маскот

### Время на компоненты (без иллюстратора)
- CSS-система и переменные: 1 день
- Базовые компоненты (cards, callout, metric): 1-2 дня
- Funnel + chart SVG: 1 день
- Маскот заглушка (потом заменим на отрисованного): 0.5 дня
- Применение к 3 кейсам: по 0.5 дня каждый
- Итого: **~6-7 рабочих дней** до готовых 3 кейсов в V2-стиле

## 13. Что показать Виктории на каждом шаге

1. **Сначала**: 3-4 SVG-мокапа — как выглядит ОДИН экран кейса (hero / этап / wow-блок / финал). Без кода. На утверждение.
2. **Потом**: один блок кейса в реальном HTML — например, hero SPB кейса по новому стилю. Сравнить с V1.
3. **Потом**: один кейс целиком (SPB как эталон V2).
4. **Потом**: остальные 2 кейса по тому же шаблону.

Не делать всё сразу. Не показывать «вот переделал 3 кейса», прежде чем согласован визуальный язык на 1 экране.
