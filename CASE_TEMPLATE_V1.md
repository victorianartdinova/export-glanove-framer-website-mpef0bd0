# CASE_TEMPLATE_V1

Шаблон кейс-страницы для агентства ГЛАВНОЕ. Маркетинг недвижимости.

Логика взята из `https://theglavnoe.com/ai-startup/` (`/root/ops/products/ai-startup-guide.html`) — пошаговая прокладка, sticky-навигация по этапам, доказательства встроены в текст, KEY TAKEAWAY-блоки, CTA после каждой смысловой секции. **Дизайн не копируем** — типографика, цвета и компоненты наследуются от тёмной палитры `theglavnoe.com` (Suisse Intl + dark cards + красный акцент `#ff3b30`).

Эталон по визуалу — `/work/case-spb-15mln/index.html` (commit `c7017cd`). Любой новый кейс делаем как клон его CSS + структуры, меняя контент и встроенные скрины.

## 0. Пять принципов шаблона

### 1. Высокая читаемость
- Один экран — одна мысль. Без полотен текста.
- Контраст: тёмный фон `#000` / `#0d0d0d`, светлый текст `#f5f5f5`, акцент `#ff3b30` только на ключевые элементы.
- Шрифт `DM Sans` (fallback Suisse Intl) — body 15-16px, line-height 1.45.
- Максимум 2 уровня нумерованных списков. Внутри карточек — только короткие пункты.
- Никаких длинных тире `—` в заголовках. В тексте — точки и запятые.

### 2. Пошаговое обучение
- 4 этапа: **Изучили → Придумали → Запустили → Получили**. Жёсткая нумерация.
- Каждый этап — большая цифра слева (`stage-num`, 84px), контент справа.
- Sticky-nav сверху с переходами по этапам (опционально на desktop).
- В каждом этапе: 1 заголовок + 1 абзац вводки + нумерованный список из 4-6 пунктов.

### 3. Визуальная аналитика
- **Funnel-блок** обязателен: 5 шагов от бюджета до сделок с цифрами и % конверсии. Узкое горлышко выделено `accent`-классом.
- **Metric-grid** 2×2 или 2×4 — главные цифры крупным шрифтом.
- **Bar-chart** (CSS, без библиотек) — динамика по неделям/месяцам.
- Цвета графиков — основной акцент + 1 нейтральный. Без 3D, градиентов, теней.

### 4. Реальные доказательства (inline в текст, НЕ галерея)
- Каждый скрин из источника — **рядом с тем шагом, который он иллюстрирует**. Не в конце страницы.
- Подпись курсивом под скрином: что показывает + источник.
- Скрины — JPG, max-width 1600px, quality 82, без оптимизации до Retina.
- Папка `assets/` внутри директории кейса. Имена осмысленные: `1_spoyler.jpg`, `4_reklama.jpg`, `final-report.jpg`.
- Каждое изображение в `<figure>` с `<img loading="lazy">` и `<figcaption>`.

### 5. Быстрый skim-reading
- **TOC** (Table of Contents) после hero, на mobile — collapsed details.
- Hero-карточка справа: одна **ключевая цифра** + KPI-сравнение зелёным.
- В каждом этапе — `KEY TAKEAWAY` блок: 1 предложение жирным с красным border-left.
- Метрика всегда читается как «N единиц · контекст»: «9 684 ₽ · CPL квал-клиента при KPI 10 000 ₽».
- Anchor-ссылки внутри TOC и в crumbs. ID на каждой `<section>`.

## 1. ОБЯЗАТЕЛЬНЫЕ БЛОКИ

Без них кейс не публикуется.

| № | Блок | Цель | Источник данных |
|---|------|------|-----------------|
| 1 | **Hero** | Главная цифра + кто/что/период | Бриф, рекламный кабинет |
| 2 | **TOC** | Skim-навигация | Авто из H2 |
| 3 | **Контекст** (3 карточки: Клиент / Период / Задача) | Кто и зачем | Бриф первого созвона |
| 4 | **Воронка** (5-step funnel) | Сквозная аналитика | CRM + рекламный кабинет |
| 5 | **Этапы 01-04** | Пошаговое доказательство | Документы проекта + скрины |
| 6 | **Метрики** (2×4 grid) | Главные цифры одним взглядом | CRM + кабинет |
| 7 | **График динамики** (1 bar-chart) | Тренд по периодам | Еженедельная разбивка |
| 8 | **CTA-блок** | Конверсия | Утверждённый CTA продукта |
| 9 | **Внешний источник** | Доверие через СМИ | URL публикации + цитата |
| 10 | **Связанные кейсы** (2 карточки) | Удержание на сайте | Портфолио |

### 1.1 Hero
- Тэг: «Кейс · [продукт]»
- H1 — единственный. Формат: «[Главная цифра] [через/для] [контекст]». Пример: «50 квал-лидов в месяц на премиум-недвижимость через Telegram Ads».
- Hero-sub: 1 предложение, 20-25 слов.
- 3 чипа метаданных: клиент · период · бюджет.
- Aside-карточка справа: ключевая цифра (72px) + label + unit + compare (зелёный).
- 2 кнопки: primary + ghost (anchor на funnel).

### 1.2 TOC
- 6-8 пунктов: «Контекст · Воронка · Этап 01 · Этап 02 · Этап 03 · Этап 04 · Метрики · Источник».
- Desktop — горизонтальный pill-bar. Mobile — `<details>` collapsed.

### 1.3 Контекст (3 карточки)
Cards-3 grid:
1. **Клиент** — отрасль, локация, чек, аудитория.
2. **Период** — даты, что входило в работу.
3. **Задача** — KPI клиента в одной фразе («квал-лид до 10 000 ₽»).

### 1.4 Воронка
- 5 шагов: Бюджет → Подписки → Обращения → Квал-лиды → Сделки.
- Каждый блок: step-n (label) + step-v (большое число) + step-l (описание).
- Узкое горлышко (главный показатель KPI) — `accent`-класс (красный border, светящийся фон).
- Под воронкой — `funnel-note`: одна строка-вывод с красным border-left.

### 1.5 Этапы 01-04
4 секции, каждая в `stage`-блоке (grid 140px / 1fr):
- Левая колонка: stage-label «Этап» + stage-num «01» (84px красный).
- Правая колонка: stage-label (название) + H3 + вводка + нумерованный `<ul>` (4-6 пунктов).
- **Inline-скрин** в правой колонке после списка — `<figure>` с подписью.

### 1.6 Метрики
- `metrics-grid` 4 колонки (на mobile — 2 / 1).
- Главная метрика — `metric-card accent` (красная цифра).
- На каждой карточке: v (38px) + l (label) + sub (mute, контекст KPI).

### 1.7 График динамики
- Один CSS bar-chart: `chart-bars` grid с `bar > .col + .lbl`.
- Высота столбцов через `style="height:NN%"`.
- Под графиком — `chart-note` с пояснением, как считались данные.

### 1.8 CTA-блок
- Тёмная карточка с градиентом.
- H2 (36px) + p (mute, 1 предложение) + 1 кнопка primary.
- **Один CTA**. Не два.

### 1.9 Внешний источник
- `src`-блок: who (издание) + what (заголовок статьи) + ссылка на статью.
- Если источник — eLama / ppc.world / vc.ru / СберПро — указываем явно.

### 1.10 Связанные кейсы
- `related`-grid 2 колонки.
- Каждая карточка: meta (тэг) + big (цифра) + ttl (1 строка) + arrow «Читать кейс →».

## 4. MOBILE-СТРУКТУРА (≤ 809px)

Базовый брейкпоинт — `max-width: 809px`. Малый — `max-width: 430px` для типографики.

### 4.1 Что упрощается
| Блок | Desktop | Mobile (≤ 809) |
|------|---------|----------------|
| Hero | grid 1.2fr / 0.9fr | stack vertical, аside-метрика снизу |
| Hero H1 | 54px | 34px (≤430 — 30px) |
| Hero metric value | 72px | 54px (≤430 — 46px) |
| Cards-3 | 3 колонки | 1 колонка |
| Funnel | 5 колонок | 1 колонка stack |
| Stage | grid 140px / 1fr | 1 колонка, stage-num над контентом, 54px |
| Metrics-grid | 4 колонки | 2 колонки (≤430 — 1) |
| Chart-bars | height 220px | height 160px |
| Related | 2 колонки | 1 колонка |
| Src-block | row | column |
| CTA-block padding | 56px | 36px 22px |

### 4.2 Что становится stack vertical
- Hero grid → stack
- Все card-grids → 1 колонка на mobile
- Stage layout → один столбец
- Src-block → column-flex

### 4.3 Что скрывается
**Ничего**. Все секции остаются. Скрываются только декоративные элементы:
- Sticky-nav этапов (desktop only)
- SVG-разделители между блоками
- Aside-figure-номера в этапах могут переходить в inline

### 4.4 Изображения
- Все `<img>` имеют `loading="lazy"` (кроме hero-фото).
- `max-width: 100%; display: block; height: auto`.
- Подписи `<figcaption>` font-size 13px на mobile (vs 14px desktop).

## 6. ТОН И СТИЛЬ

### Запрещено
- Длинное тире `—` в **заголовках**. В body-тексте — допустимо точечно.
- Слова: «эффективный», «уникальный», «индивидуальный подход», «качественный сервис», «более / свыше / порядка» без основания.
- Слово «SMM». Только «лидогенерация», «performance», «таргетинг».
- Эмодзи в текстах кейса. Только в чате с Викой, не на странице.
- Цитаты, не согласованные с клиентом.
- Имя «Денис» — `grep -i "денис\|denis"` перед коммитом должен быть пустым.

### Обязательно
- Цифры — из источника. Если оригинал говорит «503 575 ₽» — пишем «503 575 ₽», не «~500 000».
- «Квал-лид» (с дефисом) — единый термин по всем кейсам.
- Все упоминания KPI в формате «9 684 ₽ при KPI 10 000 ₽».
- Рубли — пробел перед знаком: `503 575 ₽`. Не `503575₽`.

## 7. ЧЕК-ЛИСТ ПЕРЕД ПУБЛИКАЦИЕЙ
